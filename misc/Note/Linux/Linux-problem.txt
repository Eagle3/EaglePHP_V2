### Linux小问题记录  ###

1.  service network restart 重启网卡，出现如下错误：    弹出界面eth0: 错误：没有找到合适的设备：没有找到可用于链接System eth0 的设备
	解决方式：  http://blog.51cto.com/wenzhongxiang/1433858
	
2. Linux下文件无法写入（文件无法上传）时可能原因
   A： 程序有误
   B: 没有写权限
   C: 磁盘已满，查看磁盘使用情况  df -lh
   D: Linux下inode节点已使用完，查看inode使用情况  df -i
   
3. 安装PHP扩展一般方式 
       以phpredis为例,安装其他扩展都是类似
       
	cd /usr/local/src #进入软件包存放目录
	tar -zxvf phpredis-2.2.4.tar.gz #解压
	cd phpredis-2.2.4 #进入安装目录
	/usr/local/php/bin/phpize #用phpize生成configure配置文件
	./configure --with-php-config=/usr/local/php/bin/php-config #配置
	make #编译
	make install #安装

	#安装成功后有类似下面的提示，这里是说明扩展包的目录
	Installing shared extensions:    /usr/local/php/lib/php/extensions/no-debug-zts-20121212/
	#修改php.ini 添加下面2行
	extension_dir="/usr/local/php/lib/php/extensions/no-debug-zts-20121212/"  #添加php扩展路径目录（如果已有不用再写）
	extension=phpredis.so
	
4. 	如果在虚拟机上配置了相关服务，比如： http 80 ，https 443, ftp 21, smtp 25, mysql 3306,memcached 11211, redis 6379 mongodb 27017 等等，
	可能会出现在其他机器上无法连接的问题，出现的原因可能是iptables（防火墙）给过滤了。
	解决方法：
	A: 编辑 vi /etc/sysconfig/iptables
	B: 在上面打开的文件里面加入一下语句： -A INPUT -m state –state NEW -m tcp -p tcp –dport 80 -j ACCEPT （允许80端口通过防火墙，这里以80端口为例）
	C: 需要注意的是上面这条语句不要加载文件的最后面，这样会导致防火墙启动失败，正确的应该是添加到默认的22端口这条规则的下面
	D: 重启防火墙使配置生效 service iptables restart 或者  /etc/init.d/iptables restart
	
	iptables service命令
	service iptables status    #查看iptables状态
	service iptables restart   #iptables服务重启
	service iptables stop      #iptables服务禁用
	service iptables start     #iptables服务启用
	
5. 	使用Linux安装lanmp或者其他服务时，可能会出现莫名其妙的问题，这时有可能是SeLinux在作怪。
    SeLinux 主要作用就是最大限度地减小系统中服务进程可访问的资源（最小权限原则）。
    SeLinux是Linux的一个安全策略,将以前单纯的ugo权限,变成用content来识别权限,是Linux权限系统的一次加强.
       但是,实际应用中,很多人会遇到这样那样的问题.很多编译安装软件的文档,也特意注明了,建议关闭SeLinux。
    
        解决方式
	A: 使用setup工具进入图形化关闭, 或者修改 /etc/sysconfig/selinux 文件，修改如下： SELINUX=enforcing 改为 SELINUX=disabled
	B: 防火墙打开 chkconfig iptables on
	C: service iptables restart 或者  /etc/init.d/iptables restart
	
6.  修改Linux系统默认语言
	[root@localhost ~]# echo $LANG
	en_US.UTF-8
	修改
	[root@localhost ~]# vi /etc/sysconfig/i18n
	英文：
	LANG="en_US.UTF-8"
	中文：
	LANG="zh_CN.UTF-8"
	即时生效
	source /etc/sysconfig/i18n

	id:5:initdefault:
	# 3 多用户命令行界面
	id:3:initdefault:
	
8.	Linux下某个服务设为开机启动
	第一种：修改/etc/rc.local脚本 (以Apache为例)
		A： vi /etc/rc.local
		B: 只需要把下面一行加入文件末尾即可
		   /usr/local/apache2/bin/apachectl start
		C: 或者 A B两步忽略，直接执行此步。 echo '/usr/local/apache2/bin/apachectl start ' >> /etc/rc.local   
		   
        第二种：利用chkconfig修改 (以Apache为例)。此步也同时把Apache置为Linux系统服务，可以使用service httpd start|stop|restart 等命令
        A: cp /usr/local/apache2/bin/apachectl /etc/init.d/httpd   #拷贝脚本
        B: vi /etc/init.d/httpd        		#编辑脚本，添加如下两行
           #chkconfig:2345 85 85       		#此行的345参数表示,在哪些运行级别启动,启动序号(S61);关闭序号(K61)
		   #description:Apache httpd   		#此行必写,描述服务
		C: chkconfig --add httpd            #把httpd加为Linux系统服务，可以使用service命令
		D: chkconfig --level 2345 httpd on  #把httpd加入开机启动并且设置启动级别
		E: chmod 755 /etc/init.d/httpd      #脚本添加执行权限
		F： chkconfig --list|grep httpd      #查看httpd是否加入系统服务 (httpd 	0:off	1:off	2:on	3:on	4:on	5:on	6:off )
		
		附：chkconfig命令
		chkconfig       [--add]      [--del]     [--list]      [系统服务]
　　		chkconfig       [--level/levels]      [等级代号]      [系统服务]      [on/off/reset]
		
		chkconfig --list      			    显示开机可以自动启动的服务 
		chkconfig --add httpd   	            把httpd服务加入到Linux系统服务中 
		chkconfig --del httpd   		    删除开机自动启动httpd服务
		chkconfig --level 2345 httpd on   把httpd服务加入开机启动并且设置启动级别
		
		等级代码：linux系统的运行级别。linux 将操作 环境分为以下7个等级，即
		0：关机
		1：单用户模式（单用户、无网络）
		2：无网络支持的多用户模式（多用户、无网络）
		3：有网络支持的多用户模式（多用户、有网络）
		4：保留，未使用
		5：有网络支持有X-Window支持的多用户模式（多用户、有网络、X-Window界面）
		6：重新引导系统，即重启
